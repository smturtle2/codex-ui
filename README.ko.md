# Codex WebUI

[English](./README.md) | [한국어](./README.ko.md)

이 저장소는 지금부터 완전히 새로 다시 만듭니다.

Codex WebUI는 Codex 워크플로우를 위한 로컬 우선 인터페이스 프로젝트로 다시 시작하는 중입니다. 과거 구조는 의도적으로 비워 두었고, 다음 버전은 더 작고 더 명확하고 더 정직한 기반 위에서 다시 쌓아 올릴 예정입니다.

## 현재 상태

- 저장소는 의도적으로 리셋된 상태입니다.
- 지금은 실행 가능한 앱이 체크인되어 있지 않습니다.
- 실제로 설치하고 실행할 대상이 생기기 전까지 setup 및 usage 안내는 넣지 않습니다.
- 아키텍처, 스택, 디렉터리 구조는 다시 열어 두었고 순차적으로 정리할 예정입니다.

## 왜 다시 만드는가

- 이전 구조가 더 이상 프로젝트가 가려는 방향과 맞지 않았습니다.
- 부분 수정만 반복하는 것보다 한 번 깨끗하게 리셋하는 편이 더 명확합니다.
- 문서는 삭제된 코드가 아니라 현재 저장소를 설명해야 합니다.
- 다음 버전은 최소한의 동작하는 코어에서 다시 키워야 합니다.
- 핵심 사용자 흐름에 더 분명한 경계와 더 적은 숨은 가정이 필요합니다.

## 재구축 원칙

- 문서와 구현을 항상 일치시킵니다.
- 원격 요구가 증명되기 전까지는 로컬 우선으로 접근합니다.
- 비용을 설명하지 못하는 추상화보다 명확한 구조를 택합니다.
- 추측성 아키텍처보다 관찰 가능한 사용자 흐름을 중심으로 만듭니다.
- 작게 시작하고, 코어를 먼저 만든 뒤 확장합니다.

## 첫 번째 마일스톤

1. 최소 프로젝트 스캐폴드와 개발 기준선을 다시 세웁니다.
2. 실제 출발점이 될 수 있는 가장 작은 앱 셸을 추가합니다.
3. 복잡한 UI를 다시 만들기 전에 Codex 연동 경계를 먼저 정리합니다.
4. 핵심 대화 흐름을 처음부터 끝까지 다시 만듭니다.
5. 새 구현이 자리 잡는 만큼 테스트와 문서를 다시 추가합니다.

## 아직 준비되지 않은 것

- 데모나 프로덕션 수준 빌드는 아직 없습니다.
- 지금은 유효한 실행 대상이 없으므로 Quick Start도 없습니다.
- 아키텍처는 아직 확정되지 않았습니다.
- 호환성, 보안, 배포 문서는 새 기준선이 생긴 뒤에 정리합니다.

## 기여

초기 방향, 범위, 순서에 대한 피드백을 환영합니다.

새 기준선이 만들어지기 전까지 특히 유용한 기여는 다음과 같습니다.

- 구체적인 문제 정의
- 프로젝트 구조에 대한 비판
- 결정 비용을 줄이는 작고 선명한 제안
- 이번 리셋 방향과 맞는 구현 아이디어
